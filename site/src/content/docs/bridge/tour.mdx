Everything below is one command — `kraken serve` — rendering one journal.
No bridge-only state exists: each screen is a projection of the same
append-only events written by the CLI and by agent sessions connected over
the [Model Context Protocol (MCP)](../../guides/claude-code-plugin/).

## Dark mode ships too

Light is the default face; a proper dark theme is one toggle away — same
journal, same layout, a deep-sea console for late-shift supervision:

<Shot
  src={runLiveDark}
  alt="The live-run layout in the bridge's dark theme"
  caption="The live run in dark mode. The theme is chrome; the journal is the content."
/>

<Shot
  src={channelDark}
  alt="The project channel with a proposal card, in dark mode"
  caption="The channel in dark mode, proposal card included."
/>

Pair it with `notify_url` push and the loop closes: a decision lands on your
lock screen, you tap into the bridge, decide with a note, and the fleet
resumes — [everything that didn't need you already happened without
you](../../reference/philosophy/).
